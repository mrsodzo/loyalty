## 1. Контекст и обзор архитектуры

### 1.1 Описание продукта

**Rewards Platform** — платформа лояльности, предназначенная для обработки событий из внешних систем BSS/OSS и EdTech VAS с последующей выдачей наград пользователям через сторонних провайдеров.

### 1.2 Ключевые драйверы архитектуры

| Драйвер | Описание |
|---------|----------|
| Event-driven | Обработка событий из внешних систем в режиме, близком к реальному времени |
| Multi-tenant channels | Три различных канала с разными ролями и сценариями использования |
| Integration-heavy | Интеграция с внешними провайдерами наград и платёжным шлюзом |
| AI/ML components | Субагенты с guardrails для оценки рисков и персонализации |
| Compliance & Audit | Полное журналирование всех действий, explainability |
| High availability | Строгие SLO по latency и доступности |
| Financial operations | Операции с виртуальным балансом Rewards Bank Account |

## 2. Бизнес-сценарии

### 2.1 Event → Reward (Категория 1) — Обязательный сценарий

#### Описание
Основной end-to-end поток обработки события и выдачи награды.

#### Диаграмма потока

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  Внешняя │    │          │    │          │    │          │    │          │    │          │
│  система │───▶│Ingestion │───▶│Eligibility│───▶│  Risk/   │───▶│ Rewards  │───▶│Fulfillment│
│ BSS/EdTech│    │ Service  │    │ Service  │    │  Fraud   │    │Calculation│    │ Service  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
      │               │               │               │               │               │
      │               ▼               ▼               ▼               ▼               ▼
      │         ┌─────────────────────────────────────────────────────────────────────┐
      │         │                        Kafka Event Bus                               │
      │         └─────────────────────────────────────────────────────────────────────┘
      │                                                                               │
      │                                                                               ▼
      │                                                                        ┌──────────┐
      │                                                                        │  Reward  │
      │                                                                        │ Provider │
      │                                                                        └──────────┘
      │                                                                               │
      │                                                                               ▼
      │                                                                        ┌──────────┐
      └────────────────────────────────────────────────────────────────────────▶│Subscriber│
                                                                               │   App    │
                                                                               └──────────┘
```

#### Детальные шаги

| Шаг | Компонент | Действие | SLA |
|-----|-----------|----------|-----|
| 1 | Внешняя система | Генерация события (активация, пополнение, учебная активность, покупка пакета) | — |
| 2 | Ingestion Service | Нормализация, дедупликация, обогащение метаданными | < 20ms |
| 3 | Eligibility Service | Проверка правил, сегментация, вычисление права на награду | < 30ms |
| 4 | Risk/Fraud Service | Риск-скоринг, проверка аномалий, rate limits | < 20ms |
| 5 | Rewards Calculation | Расчет баллов/кешбэк/ваучер, применение промо-кампаний | < 20ms |
| 6 | Fulfillment Service | Взаимодействие с внешним провайдером, идемпотентность | < 80ms |
| 7 | Subscriber App | Отображение результата пользователю | < 10ms |

**Общий E2E SLA:** 100-200ms

### 2.2 Sponsor Verification + Sponsored Rewards

#### Описание
Сценарий верификации спонсора и активации спонсируемых наград.

#### Диаграмма потока

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   Sponsor    │     │    Sponsor   │     │    Risk/     │     │   Rewards    │
│   Parent App │────▶│ Relationship │────▶│ Fraud/AI     │────▶│ Calculation  │
│              │     │   Service    │     │ Subagent     │     │   Service    │
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
       │                    │                    │                    │
       │                    │                    │                    │
       │                    │                    ▼                    │
       │                    │            ┌──────────────┐            │
       │                    │            │  Decision:   │            │
       │                    │            │ allow/deny/  │            │
       │                    │            │ step-up      │            │
       │                    │            └──────────────┘            │
       │                    │                    │                    │
       │                    │                    │                    │
       │                    ▼                    ▼                    ▼
       │             ┌─────────────────────────────────────────────────────┐
       │             │              Sponsored Rewards Logic                │
       │             │  - Активация семейных сценариев                     │
       │             │  - Связывание спонсор-подписчик                     │
       │             │  - Применение специальных правил                    │
       │             └─────────────────────────────────────────────────────┘
       │                                                                      │
       ▼                                                                      ▼
┌──────────────┐                                                      ┌──────────────┐
│  Subscriber  │◀─────────────────────────────────────────────────────│  Fulfillment │
│     App      │                                                      │   Service    │
└──────────────┘                                                      └──────────────┘
```

#### Решения субагента

| Решение | Описание | Действие |
|---------|----------|----------|
| `allow` | Низкий риск, верификация успешна | Активация спонсируемой логики наград |
| `deny` | Высокий риск, мошенничество | Отказ в верификации, логирование |
| `step-up verification` | Требуется дополнительная проверка | Запрос дополнительных документов/кодов |

## 3. Сервисная архитектура

### 3.1 Core Services (Обязательные)

#### 3.1.1 Ingestion Service

**Назначение:** Нормализация событий, дедупликация, обогащение метаданными.

| Характеристика | Описание |
|----------------|----------|
| Входные данные | События из BSS/OSS, EdTech VAS |
| Выходные данные | Нормализованные события в Kafka |
| Ключевые функции | Schema validation, дедупликация, enrichment |
| Масштабирование | Horizontal, по количеству входящих событий |

**API:**
```
POST /api/v1/events
POST /api/v1/events/batch
GET  /api/v1/events/{eventId}/status
```

#### 3.1.2 Eligibility Service

**Назначение:** Правила, сегментация, вычисление права на награду.

| Характеристика | Описание |
|----------------|----------|
| Входные данные | Нормализованные события из Kafka |
| Выходные данные | Решение о праве на награду |
| Ключевые функции | Rule engine, сегментация, проверка условий |
| Кэширование | Redis для правил и сегментов |

**API:**
```
POST /api/v1/eligibility/check
GET  /api/v1/eligibility/rules
POST /api/v1/eligibility/rules
```

#### 3.1.3 Risk/Fraud Service

**Назначение:** Риск-скоринг, аномалии, rate limits, step-up verification.

| Характеристика | Описание |
|----------------|----------|
| Входные данные | События, запросы на верификацию |
| Выходные данные | Риск-скор, решение allow/deny/step-up |
| Ключевые функции | ML-модели, rule-based checks, anomaly detection |
| AI Integration | Субагенты для оценки риска |

**API:**
```
POST /api/v1/risk/score
POST /api/v1/risk/verify
GET  /api/v1/risk/alerts
```

#### 3.1.4 Rewards Calculation Service

**Назначение:** Начисления/списания, промо-кампании.

| Характеристика | Описание |
|----------------|----------|
| Входные данные | Eligible события |
| Выходные данные | Расчетные награды |
| Ключевые функции | Formula engine, promo application, TTL management |
| Данные | Балансы, транзакции, промо-правила |

**API:**
```
POST /api/v1/rewards/calculate
POST /api/v1/rewards/apply
GET  /api/v1/rewards/balance/{userId}
POST /api/v1/rewards/reverse
```

#### 3.1.5 Fulfillment Service

**Назначение:** Взаимодействие с внешними провайдерами наград, идемпотентность.

| Характеристика | Описание |
|----------------|----------|
| Входные данные | Рассчитанные награды |
| Выходные данные | Статус выдачи награды |
| Ключевые функции | Provider integration, idempotency, retry logic |
| Надежность | Circuit breaker, timeout handling |

**API:**
```
POST /api/v1/fulfillment/create
POST /api/v1/fulfillment/redeem
POST /api/v1/fulfillment/cancel
GET  /api/v1/fulfillment/status/{requestId}
```

#### 3.1.6 Explainability & Audit Agent

**Назначение:** Формирование объяснений + аудит-трейл.

| Характеристика | Описание |
|----------------|----------|
| Входные данные | Все решения системы |
| Выходные данные | Объяснения, аудит-логи |
| Ключевые функции | Decision explanation, audit logging, query interface |
| AI Integration | LLM для генерации объяснений |

**API:**
```
GET  /api/v1/explain/decision/{decisionId}
GET  /api/v1/audit/logs
POST /api/v1/audit/query
```

#### 3.1.7 Ops/Admin Assistant Agent

**Назначение:** Помощник оператору с жёсткими границами прав.

| Характеристика | Описание |
|----------------|----------|
| Входные данные | Запросы операторов |
| Выходные данные | Рекомендации, данные для расследования |
| Ключевые функции | Query assistance, data summarization, alert triage |
| Guardrails | Read-only для критических данных, approval workflow |

**API:**
```
POST /api/v1/assistant/query
GET  /api/v1/assistant/recommendations
```

### 10.2 On-call и Alerting

#### 10.2.1 P0/P1 Критерии

| Severity | Критерии | Response Time |
|----------|----------|---------------|
| P0 | Полная недоступность платформы, потеря данных | 15 минут |
| P1 | Деградация сервиса, провайдер недоступен | 1 час |
| P2 | Частичные проблемы, не влияющие на бизнес | 4 часа |
| P3 | Минорные проблемы, косметические баги | 24 часа |

#### 10.2.2 Alert Categories

| Категория | Примеры | Действие |
|-----------|---------|----------|
| Availability | Service down, health check fail | Page on-call |
| Performance | Latency > SLO, queue growing | Page on-call |
| Security | PII in logs, unauthorized access | Page security team |
| Business | Reward loss, fraud spike | Page business owner |

### 10.3 Runbooks

#### 10.3.1 Провайдер недоступен

```
1. Проверить статус провайдера (status page)
2. Проверить circuit breaker state
3. Если OPEN > 5 минут:
   - Включить fallback mode (queue events)
   - Уведомить бизнес
4. Если восстановление > 30 минут:
   - Эскалация в P1
   - Рассмотреть альтернативного провайдера
```

#### 10.3.2 Очередь растёт

```
1. Проверить consumer lag в Kafka
2. Проверить ресурсы consumers
3. Scale up consumers если необходимо
4. Если lag > 10000:
   - Включить приоритетную обработку
   - Уведомить бизнес о задержках
```

#### 10.3.3 БД read-only

```
1. Проверить статус реплик
2. Переключить reads на реплики
3. Если writes критичны:
   - Эскалация в P0
   - Рассмотреть failover
```

### 10.4 Alerting на передачу запрещённой информации

|---------|-----------|--------|
| PII в логах | Regex patterns | Alert + Auto-mask |
| Паспортные данные | Pattern matching | Block + Alert |
| Номера карт | Luhn check | Block + Alert |
